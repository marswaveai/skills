# 公众号文章原型

> 写作前先判断文章属于哪种原型，每种的写法重心不同。

---

## 匹配启发

| 用户输入信号 | 推荐原型 |
|-------------|---------|
| 提到测试、尝试、购买、亲手做某事 | 调查实验型 |
| 提供具体产品/工具要评测 | 产品体验型 |
| 描述一个趋势、现象、"为什么X" | 现象解读型 |
| 有一个具体工具/Prompt 要推荐 | 工具分享型 |
| 想分享积累的经验、"N条心得" | 方法论分享型 |

---

## 调查实验型

**核心**：我替你去做了这件事。

**叙事弧**：好奇/问题出现 → 亲自下场行动 → 过程中层层发现（踩坑、意外、反转）→ 最终结论 + 读者能拿走的东西

**过往案例**：
- 淘宝上卖9块9的DeepSeek，让我看到了被折叠的魔幻世界
- 亲手给AI投毒之后，我觉得整个互联网都变成了一座黑暗森林
- 我花499找人上门安装OpenClaw

**L3-5 专项检查**：
- 有没有"亲自下场"的叙事感？读者能不能感觉到"这个人真的做了这件事"？
- 过程是否有层层递进的发现？（不是平铺直叙，是一步步揭开）
- 每一步有没有带真实反应（惊讶、无语、兴奋）？

---

## 产品体验型

**核心**：跟我一起玩。

**叙事弧**：拿到产品 → 从具体使用场景切入 → 逐个演示功能/玩法（升番逻辑：从基础到骚操作）→ 自然对比同类产品 → 真实总结判断

**过往案例**：
- 用豆包手机的这两周，我好像卷入了一场新与旧的战争
- miclaw手机Agent（从设置人设开始逐个演示）
- Seedance 2.0（按玩法分类，每个玩法配真实case）

**L3-5 专项检查**：
- 有没有真实的使用场景？（不是想象中的"如果你需要xxx"，是"我今天实际遇到了xxx"）
- 功能展示是否用了升番逻辑？（先基础功能，再进阶用法，最后骚操作）
- 有没有跟其他产品的自然对比（不强求，但有的话要真实）？

---

## 现象解读型

**核心**：你注意到了吗？背后是什么？

**叙事弧**：观察到有趣现象 → 好奇心驱动追问 → 研究/测试/挖掘 → 知识自然掉落 → 哲学/文化升维

**过往案例**：
- 三宫格图片刷屏 → 追问为什么 → 连接到十年前黑边照片 → 人生叙事需求
- AI看不到爱心 → 逐个模型测试 → 找到论文 → 时间盲视 → 格式塔心理学 → 哲学感悟
- 全世界的赛博难民，正在疯狂涌进小红书

**L3-5 专项检查**：
- 是否有清晰的"观察→好奇→研究→升维"四段推进？
- 知识掉落是否自然？（"突然想起来这个"而不是"下面我来科普"）
- 文化升维的连接是否有说服力、不生硬？

---

## 工具分享型

**核心**：我发现了一个好东西。

**叙事弧**：个人故事/痛点铺垫 → 自然引出工具 → 展示自己的使用过程（带真实结果）→ 效果展示"卧槽"时刻 → 给读者的使用建议

**过往案例**：
- 天赋挖掘Prompt → 从朋友聊天切入 → 天赋这个词的思考 → 工具展示 → 自己测试结果
- RPA+AI，才是真正能让你躺平的自动化真神

**L3-5 专项检查**：
- 有没有个人故事铺垫？（不能直接上来"推荐一个好用的xxx"）
- 工具展示过程中有没有一个让人"卧槽"的结果？
- 结尾有没有给读者清晰的"你可以这样用"行动建议？

---

## 方法论分享型

**核心**：我把压箱底的东西掏给你了。

**叙事弧**：谦逊铺垫（卸掉傲慢感）→ 用个人经历引出每一条方法 → 每条都有可执行行动 + 坦诚学习成本/失败点 → 结尾回扣所有行动点并升华

**过往案例**：
- 用AI三年的9条心得
- 分析了1.8亿个岗位后，我发现应届生们好像被AI堵在了门外

**L3-5 专项检查**：
- 开头有没有谦逊铺垫？（"我也不知道行不行""不成熟的经验"）
- 每节读完后读者手里有没有一个今天就能执行的动作？
- 有没有坦诚说明学习曲线和常见失败点？（不只画饼）
- 结尾是否回扣了所有行动点，并有升华？
