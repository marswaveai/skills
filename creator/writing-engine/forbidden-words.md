# 禁用词表与结构规范

> 本文件由所有平台共享。写作和自查时必须遵守，零容忍。

---

## 禁用词汇

| 禁用 | 原因 | 替代方案 |
|------|------|----------|
| 首先…其次…最后 | 最典型的AI套话结构，立刻暴露机器感 | 用自然的口语转场词（"说到这个"、"顺着上面的再聊聊"） |
| 综上所述 | 论文/报告腔，零口语感 | 用具体的回扣句，直接说结论 |
| 值得注意的是 | AI高频套话，读者看见就皱眉 | 直接说，删掉这四个字 |
| 不难发现 | 傲慢感，像在教学生 | 直接说观察到的现象 |
| 让我们来看看 | 典型AI主持腔 | 直接展示，不需要引导语 |
| 接下来让我们 | 同上，读PPT的感觉 | 直接切入下一段 |
| 说白了 | AI特别爱用，一出现立刻暴露 | "坦率的讲"、"其实就是" |
| 意味着什么 | AI标志性句式（"这意味着什么呢？"） | "那结果会怎样呢"、"所以呢" |
| 这意味着 | 同上，书面感过重 | "你想想看"、"也就是说" |
| 本质上 | 太学术，AI腔 | "说到底"、"其实" |
| 换句话说 | 太书面 | "你想想看"、"也就是说" |
| 不可否认 | 套话，像在绕弯子 | 直接删掉，改为正面陈述 |
| 深入探讨 | 学术腔，无意义填充 | 直接探讨，不需要宣告 |
| 全面分析 | 同上，夸大其词 | 说具体分析了什么 |
| 在当今…的时代 | 教科书开头，零信息量 | 从具体的、当下的事件切入 |
| 随着…的发展 | 同上，宏大叙事套话 | 直接说具体发生了什么 |
| 总的来说 | 等同于"综上所述"，总结腔 | 用具体的回扣句 |

## 禁用标点

| 禁用 | 原因 | 替代方案 |
|------|------|----------|
| `：`（冒号） | 产生"文件感"，过于正式 | 改用逗号，或直接不用标点 |
| `——`（破折号） | 书面腔，排版也丑 | 改用逗号或句号 |
| `""` / `""` （双引号） | 书面引用感 | 改用`「」`，或直接不加引号 |

---

## 结构反模式

以下结构模式一出现就要改写：

- **连续 bullet point 超过 3 个**：改成散文叙述，用口语化转场衔接
- **加粗文字超过 2 行**：几乎必然是过度结构化，改成普通文字或拆短
- **正文出现 markdown 标题（`##` / `###`）**：绝大多数文章不加小标题，靠口语化转场推进。唯一例外：有编号条目的方法论文章（"N条心得"类型），但即使如此也不用 markdown 格式，只用数字编号（1、2、3）

---

## 空泛工具名

永远不说"AI工具"、"某个模型"、"相关技术"。

要说具体名字，比如：Claude Code、Codex、Seedance 2.0、Deepresearch、Clawbot、DeepSeek、豆包、元宝。

---

## 教科书开头

以下几类开头绝对禁止，永远从一个具体的、当下的事件或场景切入：

- 在当今AI快速发展的时代……
- 随着技术的不断进步……
- 在探讨这个问题之前，我们首先需要了解……
- 本文将为大家介绍……
- 众所周知……
